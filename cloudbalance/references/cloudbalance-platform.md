# CloudBalance Platform Reference

> CloudBalance (https://www.cloudbalance.ai) - AWS FinOps SaaS platform.
> Analyzes AWS Cost and Usage Reports (CUR) to deliver rightsizing recommendations,
> commitment management, and AI-powered cost insights for multi-account AWS environments.

---

## What CloudBalance does

CloudBalance ingests AWS Cost and Usage Reports (CUR) from customer AWS accounts and
provides:

- **Rightsizing recommendations** - EC2, EBS, RDS, and idle resource recommendations
  sourced from AWS Compute Optimizer, pre-processed and stored for fast access
- **Commitment management** - Savings Plans and Reserved Instance tracking, coverage
  and utilization analysis, planning tools
- **Cost analytics** - Service and instance-type cost breakdowns from CUR data,
  monthly and daily granularity, multi-account views
- **AI insights** - Automated observations and trend analysis (FinChat) powered by
  AWS Bedrock with Claude
- **Automated changes** - Scheduled and immediate EC2/EBS/RDS instance type changes
  with approval workflows

---

## Data model

```
Company
  └── CompanyCloudCredentials (one per AWS account)
        └── cross-account IAM role → AWS account
              └── CUR data → ingested into CloudBalance datastore
```

- A **Company** is a CloudBalance tenant (one organization).
- A **CompanyCloudCredentials** record maps one AWS account via a cross-account IAM role.
  A company may have multiple credentials (multiple AWS accounts).
- **CUR data** is ingested from S3 on the AWS CUR publish schedule (~daily for most accounts).
- **Recommendations** are fetched from AWS Compute Optimizer via the cross-account role and
  stored in CloudBalance's database. They are refreshed weekly.
- **Commitments** (Savings Plans and RIs) are tracked daily from the CUR and AWS APIs.

## Data freshness

| Data type | Refresh cadence | Notes |
|---|---|---|
| CUR cost data | Twice daily | Follows AWS CUR publish schedule; final month data may take 24-48h |
| EC2/EBS/RDS/EKS/OpenSearch/DynamoDB recommendations | Daily | Sourced from AWS Compute Optimizer and CloudWatch |
| Savings Plans / RI commitments | Daily | Tracked from CUR and AWS APIs |
| Coverage and utilization | Daily | Calculated from CUR data |
| AI observations and insights | Daily | Generated by Bedrock during nightly refresh |

CUR data covers the most recent 13 months. Data older than 13 months is not available
through CloudBalance tools - use BCM `cost-explorer` for historical data beyond that range.

## Recommendation types

### EC2 rightsizing
- **Resize** - change instance type (family and/or size)
- **Stop** - terminate idle or consistently underutilized instances
- Source: AWS Compute Optimizer
- Savings estimate: on-demand price delta between current and recommended instance type

### EBS optimization
- **Volume type change** - e.g., gp2 to gp3 (typically 20% cost reduction)
- **Size reduction** - reduce over-provisioned volumes
- Source: AWS Compute Optimizer

### RDS rightsizing
- **Instance class change** - resize to smaller or different class
- Source: AWS Compute Optimizer
- Note: RDS changes require a maintenance window; CloudBalance can schedule them

### Idle resources
- EC2 instances, EBS volumes, RDS instances with near-zero utilization
- Recommendation: stop or terminate
- Source: CloudWatch metrics analyzed by Compute Optimizer

## Cost data structure

CloudBalance stores CUR data in its own datastore (not live CE API calls). Key dimensions:

- **group_by=service** - AWS service names (e.g., "Amazon EC2", "Amazon RDS")
- **group_by=instance_type** - EC2 instance types (e.g., "t3.medium", "m5.xlarge")
- **cost_type=unblended** - standard unblended cost (default)
- **cost_type=amortized** - amortized cost (spreads upfront RI/SP fees across the term)
- **account_id filter** - filter to a single AWS account (use the 12-digit account ID)

CUR line item types included by default: usage charges and credits.
Optionally include: refunds, taxes (rarely needed for optimization analysis).
